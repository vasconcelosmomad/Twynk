1. Deposind e criar a estrutura do projecto e os arquivos docker-compose e Dockerfile e nginex.conf
executar Tente executar o comando para criar o projeto:
docker compose run --rm twink composer create-project laravel/laravel .
Depois que o comando docker compose run --rm twink composer create-project laravel/laravel . terminar, você poderá iniciar seus contêineres com:
docker compose up -d
E o aplicativo Laravel estará acessível em http://localhost:8080.

Obrigado por fornecer a listagem do diretório. O erro Project directory ... is not empty ocorre porque o Composer precisa de um diretório vazio para criar o projeto, mas o diretório src já contém a pasta storage.

Além disso, como você pode ver na listagem, o diretório src e seu conteúdo pertencem ao usuário root. Isso impedirá que o Composer crie os arquivos do projeto.

Para resolver isso, primeiro vou alterar a propriedade do diretório src para o seu usuário (vasconclos). Isso exigirá sua aprovação, pois usa sudo.

/home/vasconclos/develop/twink_app
Terminal
sudo chown -R vasconclos:vasconclos /home/vasconclos/develop/twink_app/src
Ótimo. Agora que você é o proprietário do diretório src, vou remover o conteúdo dele para que o Composer possa criar o projeto.
